# 経費精算システム テスト仕様書

## 1. 概要

本書は経費精算システムのテスト仕様を定義する。
テストは **単体テスト**・**統合テスト**・**E2Eテスト** の3層で構成し、以下の10機能を対象とする。

| # | 対象機能 |
|---|----------|
| 1 | Microsoft ログイン |
| 2 | 一覧画面表示 |
| 3 | 新規登録 |
| 4 | 編集 |
| 5 | 削除 |
| 6 | フィルター |
| 7 | ソート |
| 8 | ページネーション |
| 9 | 合計金額表示 |
| 10 | ログアウト |

### テスト環境

| 項目 | 内容 |
|------|------|
| フレームワーク (単体・統合) | Jest + React Testing Library |
| フレームワーク (E2E) | Playwright |
| DB | SQLite（テスト用ファイル） |
| 認証 | NextAuth.js v4 + Microsoft Entra ID（E2Eではモック使用） |

## 5. テスト優先度

| 優先度 | テスト範囲 | 理由 |
|--------|-----------|------|
| 高 | 認証（UT-API-001, IT-021, E2E-004） | セキュリティに直結 |
| 高 | CRUD 操作（UT-API-011〜052, IT-001〜003） | コア機能 |
| 高 | バリデーション（UT-API-012〜020, UT-CL-001〜008） | データ品質の確保 |
| 中 | フィルター・ソート（E2E-050〜066） | ユーザビリティ |
| 中 | ページネーション（E2E-070〜076） | 大量データ時の操作性 |
| 中 | 合計金額（UT-CL-010〜013, E2E-080〜085） | 業務上の正確性 |
| 低 | ファイルアップロード（UT-API-060〜067） | 補助機能 |

|------|------|-----------|------|
| 一覧表示テスト | 15件 | 各ステータス混在 | ページネーション検証用 |
| フィルターテスト | 各カテゴリ2件以上 | 各ステータス2件以上 | 絞り込み検証用 |
| 合計金額テスト | 当日2件、当月5件、当年10件 | - | 集計検証用 |
| ソートテスト | 5件以上 | - | 件名・金額・日付がバラバラ |
